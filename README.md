# بوت الميوزك

## 1) المتطلبات
- Python 3.10 فأكثر
- FFmpeg مثبت على الجهاز/السيرفر ومتاح في الـ PATH
  - ويندوز: نزّل من ffmpeg.org وضيفه للـ PATH
  - لينكس (Ubuntu/Debian): `sudo apt install ffmpeg`
  - ماك: `brew install ffmpeg`

## 2) تثبيت المكتبات
```
pip install -r requirements.txt
```

## 3) إعداد البوت في Discord Developer Portal
1. روح https://discord.com/developers/applications وسوي تطبيق جديد
2. من تبويب Bot: فعّل صلاحية **MESSAGE CONTENT INTENT** (مهم جداً وإلا البوت ما يقرأ الرسائل)
3. انسخ التوكن (Token)
4. من تبويب OAuth2 → URL Generator: اختر `bot` وصلاحيات:
   - Read Messages/View Channels
   - Send Messages
   - Connect
   - Speak
5. افتح الرابط الناتج وضيف البوت لسيرفرك

## 4) إعداد ملف .env
انسخ `.env.example` وسمّه `.env` وعبّي القيم:
```
DISCORD_TOKEN=التوكن_تبعك
VOICE_CHANNEL_ID=ايدي_الروم_الصوتي
```
للحصول على ايدي الروم الصوتي: فعّل "Developer Mode" من إعدادات الديسكورد (Advanced)،
بعدين اضغط يمين على الروم الصوتي واختر "Copy Channel ID".

## 5) التشغيل محلياً (اختياري للتجربة)
```
python bot.py
```

## 6) النشر على Railway (تشغيل دائم 24/7)

### أ) رفع المشروع على GitHub
1. سوي مستودع (repo) جديد على GitHub
2. ارفع كل ملفات المشروع (bot.py, requirements.txt, nixpacks.toml, Procfile, railway.json, .gitignore)
3. **لا ترفع ملف `.env`** — فيه التوكن السري، و`.gitignore` مضبوط أصلاً يستثنيه

### ب) ربط GitHub بـ Railway
1. ادخل https://railway.app وسجل دخول
2. اضغط **New Project → Deploy from GitHub repo**
3. اختر المستودع اللي رفعته
4. Railway بيكتشف تلقائياً إنه مشروع بايثون (بفضل ملف nixpacks.toml بيثبت FFmpeg تلقائياً)

### ج) إضافة المتغيرات السرية
من تبويب **Variables** بمشروعك في Railway ضيف:
```
DISCORD_TOKEN = التوكن_تبعك
VOICE_CHANNEL_ID = ايدي_الروم_الصوتي
```
(هذي تحل محل ملف .env — ما تحتاج ترفعه أصلاً)

### د) التشغيل
Railway بيقرأ ملف `railway.json` تلقائياً ويشغل الأمر `python bot.py`،
وبفضل `restartPolicyType: ON_FAILURE` إذا صار خطأ أو انقطاع، البوت يعيد التشغيل من نفسه.

## 7) تشغيل أكثر من بوت (بوت لكل روم صوتي)

لو عندك عدة رومات صوتية وتبي بوت مستقل لكل روم (مثل: Silence, Stray, Moon, Dusk, Stars...)،
هذا الكود يدعم هذا الشي، بس لازم تتبع الخطوات بالضبط عشان ما يصير تعارض:

### أهم قاعدة
**كل بوت لازم يكون تطبيق (Application) منفصل تماماً في Discord Developer Portal**، يعني توكن مختلف
لكل بوت. ما تقدر تستخدم نفس التوكن لأكثر من نسخة شغالة بنفس الوقت (الديسكورد ما يسمح).

### الخطوات لكل بوت (كررها 6 مرات لـ 6 بوتات)
1. سوي تطبيق جديد بـ Developer Portal (اسم مختلف لكل بوت، مثلاً: `LS-Stray-Bot`)
2. فعّل Message Content Intent، وانسخ التوكن
3. ضيفه للسيرفر بنفس رابط الدعوة العادي (كل البوتات تنضم لنفس السيرفر)
4. بـ Railway: سوي **Service منفصل** لكل بوت (نفس الـ GitHub repo، بس Service جديد لكل بوت)
5. بكل Service ضبط المتغيرات الخاصة فيه:
   ```
   DISCORD_TOKEN = توكن البوت رقم كذا
   VOICE_CHANNEL_ID = ايدي روم الصوت المخصص لهذا البوت (مختلف لكل بوت)
   ```

### ليش ما فيه تعارض بين البوتات؟
- كل بوت عملية (process) منفصلة تماماً، وذاكرته (القائمة، التكرار...) مستقلة عن البوتات الثانية
- الكود مضبوط إنه **يتجاهل أي أمر إلا إذا الشخص اللي كتبه جالس فعلياً بنفس الروم الصوتي المخصص لهذا البوت**
  يعني لو أنت قاعد بروم "Stray" وكتبت `ش اغنية`، بوت "Stray" بس يشتغل، والباقي يتجاهلون الرسالة تلقائياً
- كل بوت يتصل ويضل فقط بالروم اللي محدد له في `VOICE_CHANNEL_ID` الخاص فيه

### تنبيه بسيط
بما إن ما فيه بريفكس، أي رسالة تبدأ بحرف "ش" (حتى لو كلمة عادية زي "شكراً") بتعتبر أمر تشغيل
من نفس البوت المخصص لروم الشخص. هذا افتراضي حسب طلبك الأول، خله ببالك وأنت تستخدمه.



عند التشغيل، البوت راح يدخل تلقائياً للروم الصوتي المحدد ويضل فيه بشكل دائم،
وإذا انفصل لأي سبب راح يرجع يتصل تلقائياً كل 15 ثانية.

## الأوامر (بدون بريفكس، تكتب في أي روم نصي يشوفه البوت)
| الأمر | الوظيفة |
|---|---|
| `ش <اسم الأغنية أو رابط يوتيوب>` | بحث وتشغيل من يوتيوب (يضيفها للقائمة لو فيه أغنية شغالة) |
| `س` | سكيب للأغنية الحالية |
| `تكرار` | تفعيل/إيقاف تكرار الأغنية الحالية |
| `قدم` أو `قدم 10` أو `قدم 50` | تقديم الأغنية بالثواني المطلوبة (افتراضي 10 ثانية) |
| `مفضلتي` | يعرض قائمة الأغاني المحفوظة بمفضلتك عند هذا البوت |

## واجهة البوت (البطاقة المولّدة)
مع كل أغنية تطلع صورة (بطاقة) مولّدة تلقائياً فيها:
- خلفية مموّهة من صورة الأغنية + صورة الألبوم واضحة على الجنب
- عنوان الأغنية واسم القناة
- شريط تقدّم يتحدث **تلقائياً كل 10 ثواني** أثناء التشغيل
- أيقونة تشغيل/إيقاف تزيينية توضح الحالة الحالية

وتحتها صفّين من الأزرار الحقيقية:
- **الصف الأول:** 🔁 تكرار / 🔊 رفع الصوت / ⏯️ تشغيل-إيقاف / 🔉 خفض الصوت / ❤️ إضافة للمفضلة
- **الصف الثاني:** ⏭️ سكيب / ⏹️ إيقاف نهائي ومسح القائمة

جميع الأزرار (عدا ❤️) تشتغل فقط لو الشخص جالس بنفس الروم الصوتي المخصص لهذا البوت.

## المفضلة (❤️)
كل شخص له قائمة مفضلة خاصة فيه (محفوظة بملف `favorites.json`)، يقدر يشوفها بأمر `مفضلتي`.

⚠️ **ملاحظة مهمة:** بما إن كل بوت (روم) هو Service منفصل على Railway، فكل بوت عنده ملف
مفضلة خاص فيه ومنفصل عن باقي البوتات. يعني لو فضّلت أغنية وأنت بروم "Stray"، ما راح تشوفها
لو سألت بوت "Moon" عن مفضلتك. لو تبي مفضلة موحّدة بين كل البوتات، هذا يحتاج قاعدة بيانات
مشتركة (مثل قاعدة بيانات خارجية) وهذا خارج نطاق الإعداد الحالي — قولّي لو تبي نضيفه لاحقاً.

⚠️ **ملاحظة ثانية:** نظام ملفات Railway مش مضمون يضل محفوظ بعد كل Redeploy (يعني ممكن تنمسح
المفضلة لو أعدت النشر). لو تبي المفضلة تضل محفوظة دائماً، ضيف **Volume** من إعدادات
الـ Service بـ Railway (Settings → Volumes) واربطه بمسار مثل `/data`، وحط بالمتغيرات:
```
FAVORITES_PATH = /data/favorites.json
```


## الخط العربي (مهم لظهور النصوص صح على البطاقة)
لازم تحط ملف خط عربي (.ttf) بالمسار `assets/fonts/arabic.ttf` — التفاصيل والخطوات
موجودة بملف `assets/fonts/READ_ME_FIRST.txt` داخل المشروع. بدون هذا الملف، النصوص العربية
على الصورة ما راح تطلع بشكل صحيح.


## ملاحظات مهمة
- هذا بوت ذاتي الاستضافة، لازم يضل يشتغل عندك (جهاز أو VPS) عشان يضل متصل بالروم 24/7.
- تحميل الصوت من يوتيوب عبر yt-dlp يخضع لشروط استخدام يوتيوب، استخدمه على مسؤوليتك.
- لو صار خطأ "Privileged intent" تأكد إنك فعّلت MESSAGE CONTENT INTENT من خطوة 3.
